# Editorial review

The article editor keeps content in the main column and validation evidence beside it on wide screens. Smaller screens stack the validation panel below the editor.

Review quality gates, source and claim records, originality warnings, internal links, external references, image licensing, static HTML, and feed eligibility. Clicking a claim opens its supporting source and status. Publishing requires explicit editorial confirmation and every server-side gate.

Manual writing, source review, image import, scheduling, and static publishing work while Groq is disabled. Hidden prompts, credentials, private notes, and raw provider responses are never exposed.
